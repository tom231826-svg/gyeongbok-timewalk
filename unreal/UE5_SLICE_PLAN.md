# 언리얼 5 수직절편 — "1888년, 근정전 마당" (AC급 한 컷)

> 작성 2026-07-06. 목표는 궁 전체가 아니라 **마당 하나**: 근정전 권역을 어쌔신 크리드급 화면으로.
> 산출물 = ①4K 스틸 6장 ②30~60초 시네마틱 영상 — 피칭·지원사업·스폰서 접촉의 무기.
> 웹 버전(타임워크)은 별개 제품으로 계속 유지 — 이건 "화질의 천장을 보여주는" 병렬 트랙.

## 왜 되는가 (전제 검증)

- 장비: Apple M5 · 16GB RAM · Metal 4 · 디스크 310GB — UE5 에디터 구동 가능(16GB가 병목, §리스크 참조)
- 에셋: 국가유산청 실측 스캔 원본 보유 — 마당 둘레(서·동월랑, 서·동행각, 사정문) 5동 + 원경 경회루.
  원본 삼각형 밀도(예: 서월랑 920만)는 나나이트(무제한 폴리곤 기능)에 그대로 먹일 수 있는 급.
- 파이프라인: 임포트→배치→라이팅→카메라 전 과정을 **파이썬 스크립트로 준비**(unreal/scripts/) —
  준석님은 에디터에서 스크립트 실행 버튼만 누르면 되는 구조로 최소화.

## 페이즈

### P0 — 준석님 액션 (기계가 못 하는 것)
| 언제 | 무엇 | 비고 |
|---|---|---|
| 오늘/내일 | Epic Games 계정 생성 → Epic Games Launcher 설치 → **UE 5.7 설치**(현 안정판, Apple Silicon 네이티브) | 다운로드 수십 GB — 자기 전에 걸어두기. 로그인 필수 |
| 월요일 | digital.khs.go.kr 이용신청: **근정전 + 근정문 + 광화문** 3건 | 근정문 추가됨(마당 정문). 신청 문구는 MASTER_PLAN §5 톤으로 |
| 아무 때 | Sketchfab 무료 가입 → KHS_Asset 계정에서 **어좌(CC-BY 확인됨)·품계석** 다운로드(glTF) | 소품 디테일용. 5분 |
| 설치 후 | UE에서 프로젝트 생성(Blank·No Starter Content, 이름 GyeongbokSlice) → Edit>Plugins에서 **Python Editor Script Plugin** 켜기 → 재시작 | 이후는 스크립트가 일함 |

### P1 — 에셋·스크립트 준비 (AI, 진행 중)
- [x] 배치 좌표 `layout_1888.json` (웹 빌드에서 검증된 북궐도형 축 좌표 재사용)
- [ ] `assets_manifest.json` — 원본 FBX 9개 검측(삼각형 수·텍스처 해상도·바운딩박스) ← 소넷 진행 중
- [ ] `scripts/s1~s4` — 임포트/배치/1888 라이팅/시네카메라 파이썬 ← 오푸스 진행 중
- [ ] macOS Nanite·Lumen 제약 확정 ← 소넷 리서치 진행 중 (결과에 따라 s3에 폴백 세팅 반영)

### P2 — 퍼스트 라이트 (설치 완료 직후, 1~2시간)
1. 에디터 파이썬 콘솔에서 `s1_import_assets.py` (임포트) → `s2_layout_level.py` (배치)
   ⚠️ 리서치 확정: **한 번에 1~2동씩 순차 임포트**(16GB 램 보호 — s1이 순차 처리하도록 설계됨), 라이트맵 UV 생성 끔, 임포트 후 나나이트 빌드 시간 있음(대형 스캔은 동당 수 분)
2. 마당 둘레 4동 + 사정문이 실측 그대로 서 있는 첫 화면 확인 → 스크린샷 1호
3. 문제(스케일·회전·텍스처)가 보이면 그 화면을 저에게 → 스크립트 수정 → 재실행 (이 반복이 기본 작업 루프)

### P3 — 1888년의 빛 (저녁 1~2회)
- `s3_scene_1888.py`: 노을 태양·하늘·안개·후처리 + Lumen 전역조명 → "사진 같은" 순간이 여기서 옴
- 16GB 대응: 에디터 스케일러빌리티 Medium으로 작업, 촬영 때만 Cinematic

### P4 — 근정전 투입 (승인 도착 시)
- layout의 `PENDING`에 경로만 채우고 s1·s2 재실행 → 주인공 등장
- 지연 시 플랜B: 월대+마당 중심 구도로 촬영(둘레 회랑이 이미 주인공급 디테일)

### P5 — 마감 촬영 (주말 1회)
- 박석 바닥 머티리얼(스캔 텍스처 타일화), 어좌·품계석·드므 소품, 깃발·차일 약간
- `s4_camera_shots.py` 시네카메라 3구도 + High-Res Screenshot(4K) / Movie Render Queue(영상)
- 성공 판정: **스틸을 AC 스크린샷 옆에 놓고 "같은 장르"로 보이면 합격**

## 일정·비용 추정 (정직하게)

## 이 트랙과 웹 트랙의 관계
- 웹(타임워크) = 시대 6개·접근성·공유 — **제품**. 언리얼 = 화질 천장 증명 — **피칭 무기**.
- 장기적으로 "문헌 기반 시대 수정"은 블렌더 클린 마스터에서 하고, 웹·언리얼 양쪽으로 내보내는 단일 원본 구조(자세한 건 다음 플랜).
