# 경복궁 타임워크 (가칭) — 총괄 마스터 플랜

> 작성: 2026-07-04 · 작성 주체: Fable 5 (총괄 설계) · 실행 주체: 하위 모델 세션들
> 프로젝트 한 줄 정의: **"로드뷰처럼 1인칭 시점으로 경복궁을 걸어다니되, 슬라이더 하나로 시대(세종대 → 고종 중건기 → 일제강점기 → 현재)를 전환하며 같은 자리의 다른 모습을 보는 웹 서비스"**

---

## 0. 결론 요약 (타당성 판정)

**조건부 실현 가능. 1인 사업자 기준으로도 MVP는 여름 안에 만들 수 있다.**

근거 3가지 (2026-07-04 웹 리서치로 검증됨):

1. **재료가 공짜로 다 나와 있다.** 국가유산청이 경복궁 건물의 정밀 3D 스캔 에셋 3만 4천여 개를 무료 공개 중이고(digital.khs.go.kr), 고종대 경복궁 전체 배치도인 '북궐도형' PDF, 일제강점기 유리건판 사진 800여 장(e뮤지엄)까지 전부 **공공누리 제1유형(출처표시만 하면 상업 이용·변형 가능)**으로 확인됨.
2. **정확히 이 컨셉의 상설 서비스는 국내외에 없다.** 정부 프로젝트(돈의문 AR, 창덕ARirang, 헤리버스 공존, 타임스코프)는 전부 한 지점짜리이거나 몇 달짜리 행사로 끝났음. 로마·파리의 해외 사례도 키오스크/투어 기기 방식이라 "웹에서 상시로 시대를 오가며 걷는" 통합 플랫폼은 빈틈.
3. **비용 구조를 뒤집을 수 있다.** 게임처럼 실시간 3D로 만들면 수억 원 프로젝트지만, **카카오 로드뷰 방식(고정 지점 360° 파노라마 + 지점 간 이동 + 시대 전환)**으로 만들면 미리 렌더링한 이미지만 웹에 띄우면 되므로 1인+AI로 가능한 규모가 됨.

단, 조건이 있다: §4 리스크(고증 자문, 정부의 동일 방향 움직임, 촬영 허가 타이밍)를 지켜야 함.

## 1. 제품 컨셉

### 1.1 핵심 UX (로드뷰 + 타임슬라이더)
- 화면: 360° 파노라마 뷰어 (마우스/터치로 둘러봄, 로드뷰와 동일)
- 바닥의 화살표로 다음 지점 이동 (광화문 → 흥례문 → 근정문 → 근정전 …)
- 화면 하단 **시대 슬라이더**: 같은 자리에서 시대를 드래그하면 크로스페이드로 장면 전환
  - 예: 근정전 앞마당에서 슬라이더를 당기면 → 1888년(고종 중건 전성기, 문무백관 조회) → 1929년(일제강점기, 앞에 총독부 청사) → 2026년(현재)
- 각 시대·지점마다 짧은 해설 카드 (실록 기록 인용 등)

### 1.2 시대 구성 (자료 확보 수준 기준)
| 시대 | 연대 설정 | 근거 자료 | 정확도 | 투입 단계 |
|---|---|---|---|---|
| 고종 중건기 | 1888년경 (약 500동 전성기) | 북궐도형(전체 배치 실측도) + 현재 복원 건물 상당수가 고종대 원형 | ★★★ 가장 정확 | **MVP** |
| 일제강점기 | 1929년경 (조선박람회, 총독부 청사) | 유리건판 사진 800여 장 | ★★★ 사진 실물 | **MVP** |
| 현재 | 2026 | 실사 360° 촬영 + 국가유산청 3D 스캔 | ★★★ | **MVP** |
| 세종대 | 1440년경 | 조선왕조실록 기록 (도면 없음 → **"추정 복원" 라벨 필수**, 전문가 자문 필요) | ★☆☆ | Phase 2 |
| (보너스) 폐허기 | 1592~1865 (임진왜란 소실 후 273년) | 문헌 + 분위기 연출 | 연출 성격 | Phase 3 옵션 |

핵심 트릭: **근정전은 1867년 고종대 원형 건물이 지금도 서 있다.** 즉 "현재 모습의 3D 스캔"이 곧 고종대 장면의 뼈대가 되고, 북궐도형을 보고 지금은 사라진 주변 건물을 덧붙이고 현대 요소(관광객, 안내판)를 지우면 고종대 장면이 완성된다. 제일 정확한 시대가 제일 싸게 만들어지는 구조.

### 1.3 MVP 범위 (의도적으로 아주 작게)
- **지점 1개**: 근정전 앞마당 (경복궁의 아이콘, 3개 시대 대비가 가장 극적)
- **시대 3개**: 고종 중건기 / 일제강점기 / 현재
- **웹 브라우저에서 열리는 데모 1개** — 앱 설치 없음, 링크 하나로 공유
- 목표 용도: ① 국가유산 디지털 콘텐츠 경진대회 출품 ② 정부·스폰서 접촉용 "보여줄 것"

## 2. 기술 설계 (하위 모델 실행용)

### 2.1 아키텍처 원칙
- 실시간 3D 아님. **사전 렌더링된 360° 파노라마 이미지(equirectangular)**를 웹 뷰어로 제공.
- 뷰어: Marzipano / Pannellum / Three.js 파노라마 중 택1 (전부 무료 오픈소스). 시대 전환은 동일 카메라 위치의 파노라마 2장 크로스페이드.
- 호스팅: 정적 웹 (비용 월 0~수천 원 수준).
- 모든 자료의 출처표시(공공누리 1유형 의무)를 페이지 하단 크레딧에 자동 포함.

### 2.2 시대별 장면 제작 파이프라인
1. **현재(2026)**: 현장에서 360° 카메라 실사 촬영 1일 (§4.3 촬영 규정 확인 후). 대안: 국가유산청 3D 에셋을 블렌더에 조립해 렌더.
2. **고종 중건기(1888)**: digital.khs.go.kr에서 근정전·근정문·행각 3D 에셋 다운로드 → 블렌더에서 씬 조립 → 북궐도형 대조로 배치 검증 → 사라진 요소 보강, 현대 요소 제거 → 카메라를 근정전 앞마당 눈높이(1.6m)에 놓고 360° 렌더.
3. **일제강점기(1929)**: 유리건판 사진 기반. 방법 A: 사진들을 참조해 3D 씬 변형(총독부 청사 매스 모델 추가, 훼철 반영) 후 렌더. 방법 B(더 빠름): 대표 사진 몇 장을 파노라마에 합성하되 "사진 기반 재구성"임을 명시.
4. AI 이미지 생성 도구로 디테일 보강 가능하나, **AI 생성/추정 부분은 반드시 화면에 표기** (고증 신뢰가 이 제품의 생명).
