## 2. 빌드·실행 (복붙)

```bash
cd ~/projects/gyeongbok-timewalk/prototype/src
node --check app.js          # 문법 검사
python3 build_v2.py          # → ../index.html(완전판), ../artifact.html(배포판)
open ../index.html           # 로컬 실행(파일 하나로 완결, 서버 불필요)
```

새 FBX(예: 근정전 본채 승인 후)를 웹용으로 바꿀 때:
```bash
/Applications/Blender.app/Contents/MacOS/Blender -b \
  -P ~/projects/gyeongbok-timewalk/prototype/src/convert_csp.py -- \
  "<FBX절대경로>" ~/projects/gyeongbok-timewalk/prototype/assets/scans_csp <이름> 70000
```
출력 로그의 `[MAT] material=... file=...` 줄이 앱에 넣을 재질↔텍스처 매핑이다.
그 다음 `app.js`의 `SCANS` 배열에 항목 추가(+`build_v2.py`에 토큰 치환 추가, 기존 3개 패턴 복붙).

## 4. 왜 이렇게 되어 있나 (결정 이력 — 뒤집기 전에 읽기)

1. **웹 아티팩트의 보안정책(CSP)이 blob URL을 전면 차단**한다(fetch·worker·img 전부). 그래서:
   - Draco 압축 금지(디코더가 wasm+worker 필요) → 무압축 GLB
   - GLB에 텍스처 내장 금지(GLTFLoader가 blob으로 풀음) → 텍스처를 data URI로 앱에서 직접 주입(`dataTexture()`, `flipY=false` 필수)
   - GLB 자체는 `loader.parse(arraybuffer)`로 넘겨서 fetch 자체가 없음
   - 로컬 파일(index.html)엔 CSP가 없으므로 Draco판(`assets/scans/`)도 쓸 수 있지만, 빌드 이원화를 피하려고 현행은 양쪽 다 무압축판 사용
2. **블렌더 5.1의 구형 FBX 임포터(`import_scene.fbx`)는 이 스캔 파일에서 크래시**한다. 반드시 신형 `bpy.ops.wm.fbx_import` 사용(이미 스크립트에 반영). 헤드리스에서 `read_factory_settings`도 불안정 → 기본 씬 오브젝트 삭제로 대체.
   - 정확한 표현: Blender는 **FBX→GLB 변환·데시메이션·텍스처 추출 파이프라인**에 쓰였다. 최종 화면 렌더링은 Blender가 아니라 브라우저의 **three.js/WebGL**이 한다. GLB 내부 `asset.generator`는 `Khronos glTF Blender I/O v5.1.20`로 확인됨.
3. **스캔 좌표는 실측의 약 1/100 스케일 + 원점 제각각** → 앱이 로드 후 바운딩박스로 자동 맞춤(긴 축을 목표 길이에 맞춰 스케일 → 회전 → 바닥 y=0 → 목표 좌표 평행이동). 목표 좌표는 `SCANS[].target`.
4. **스캔 ↔ 절차 생성 회랑 교체는 인덱스 정렬**: `SCANS[i]`가 로드되면 `procSwap[i]`(같은 자리의 절차 생성물)를 시대 3~5에서 숨김. `SCANS` 순서와 회랑 생성 loop의 `swap:true` 순서(남좌→남우→동쪽)를 바꾸면 안 된다.
5. **충돌(벽 통과 판정)은 시대별**: `colliders[].eras` — 1929엔 총독부가 막고, 폐허기엔 그 자리가 뚫려 있다. 새 건물 추가 시 `place(obj, x, z, eras)`에 시대 배열을 정확히 주면 자동 처리.
6. **고증 정직성 = 제품 원칙**: 시대마다 우상단 뱃지(고증 A/B/C), 추정 요소는 라벨. 세종·태조 시대를 "그럴듯하게" 업그레이드하지 말 것 — 자문단 검수 전엔 의도적으로 양식화 유지(스캔과의 시각 대비가 곧 고증등급 표현).
7. 배치 근거는 북궐도형 중심축 근사(광화문0 → 흥례문-135 → 영제교-176 → 근정문-228 → 근정전-318 → 사정전-432 → 강녕전/교태전, 경회루 서측 -128/-352). 정밀 대조는 `data/01_documents/북궐도형_건축.pdf`.

## 8. 실패 분기

- **블렌더가 죽으면** → 신형 임포터(`wm.fbx_import`)를 쓰고 있는지부터 확인. macOS "예기치 않게 종료" 팝업은 헤드리스 크래시 잔재이니 무시하고 로그를 볼 것
- **빌드 후 아티팩트에서 스캔이 안 보이면** → 콘솔에서 blob/fetch 오류 검색. 있으면 누군가 Draco나 내장 텍스처를 되살린 것(§4-1 위반)
- **GLB가 로컬에선 보이는데 배포판에서 텍스처가 하얗게 나오면** → `dataTexture`의 `flipY=false` 확인, 재질명↔`SCANS[].tex` 키 매칭(`[MAT]` 로그) 확인
- **아티팩트 업로드가 용량으로 거부되면** → `build_v2.py`의 `build(include_haeng=False)`처럼 스캔을 선별 포함(패턴 있음)
- **이 문서와 실물이 어긋나면** → 실물이 진실. `git`은 아직 없음 — 첫 작업으로 `git init` + 커밋 추천
