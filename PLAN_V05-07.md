## 문제 1 — "시대가 달라져도 스캔한 건물이 그대로다"

### 원인 진단
실측 스캔은 **2020년대 복원 상태가 텍스처에 구워진 것**(색 바랜 단청, 때, 얼룩). 조사 결론:
**조명·색보정으로는 구워진 낡음을 못 지운다.** 아무리 1888년 노을빛을 비춰도 때는 때로 보인다.
스캔을 늘릴수록 이 문제는 커진다(진짜가 많아질수록 "전부 2026년"처럼 보임).

### 해법 (조사 검증됨 — Rome Reborn·노트르담·AC유니티 방식의 우리화)

**P1-A. 시대별 텍스처 파생 (핵심·근본 해결)** — 공수 중
- 빌드 파이프라인에 "텍스처 시대 변형" 단계 추가. 같은 GLB(형태 불변)에 **시대별로 다른 텍스처를 주입**.
  우리 구조가 이미 텍스처 분리 주입식(SCAN_DATA.tex)이라 구조 변경 없이 텍스처 세트만 늘리면 됨.
- 방향을 뒤집는 게 핵심 절약: **"1888 새 궁궐" 텍스처를 기준으로 만들고, 2026은 원본(낡음) 그대로** 쓴다.
  때를 지우는 건 어렵지만(1회 작업), 이후 시대별 변형은 싸다.
- 단계: ① 자동화 1차 — 채도·명도 보정, 단청 색범위(청록·주홍) 선택 재채색, 때·얼룩 감쇠(이미지 처리 스크립트, 텍스처당 수 분) ② 선택 2차 — 눈에 띄는 부위만 수작업 리터칭.
- 1929 = 1888 파생본에서 약간 퇴색 + 채도 감소.
- 용량 대응: 시대 파생본은 1024px로(원본 2048 유지) → 배포판 증가 최소화. 그래도 초과 시
  중간 단계로 "셰이더 색변환 근사"(시대별 채도·색조 행렬만 적용, 텍스처 1벌) 폴백.

**P1-B. 시대별 세트드레싱 (인지 효과 최대 — 업계 공통 결론)** — 공수 중
- 조사 결론: 시대감을 가장 빨리 인지시키는 건 재질보다 **소품**이다.
- 시대별 시그니처 소품 각 3~5종만:
  - 1888: 취두 위 잡상, 드므(청동 방화수 독 — 국가유산청 무료 에셋 존재), 차일 기둥, 등롱, 궁 앞 깃발
  - 1929: 박람회 배너(있음) + 가설 홍보탑 + 전신주·전선 + 벤치
  - 2026: 관람 로프·안내판·소화기함, 관광객 실루엣
  - 폐허: 잡초(있음) + 무너진 담 더미 + 그을음 데칼
- 광화문·근정문·근정전 반경 우선(플레이어 체류 90%).

**P1-C. 1929 화면 질감 (양념)** — 공수 저
- 비네트(정적·안전) + 세피아 톤 약하게 + 필름 그레인은 **이동 중 강도 자동 감소**(멀미 예방 — 조사 명시).

### 판정 기준 (v0.5 완료 조건)
같은 자리(근정문 앞) 시대별 스크린샷 6장을 섞어 놓고 **1초 안에 어느 시대인지 구분되는가.**

## 문제 2 — "직접 지은 건물들 퀄리티가 구리다"

### 원인 진단
상자+지붕 지오메트리와 수제 그림 텍스처는 실측 스캔 옆에서 대비돼 더 싸 보인다.
조사에서 배제 확정: **AI 3D 생성(궁궐급 품질 미달), BIM 부재 키트(공수 과다), 상업 한옥팩(민가 스타일 — 궁궐 고증 불일치).**

### 해법 (펄어비스·업계 관행의 저비용 번역)

**P2-A. 스캔에서 훔친 타일 텍스처 (치트키 — 조사로 표준 기법임을 확인)** — 공수 중, 효과 최고
- 확보한 스캔 9동의 텍스처에서 **기와 골·처마 단청·사고석 벽·창호** 부위를 잘라 타일러블 텍스처로 제작.
- 1차는 무료 도구(이미지 처리 스크립트로 크롭+이음새 보정), 필요시 2차로 Substance Sampler Auto Tiling(유료).
- 이걸 절차 생성 건물 전부에 입히면 **톤이 스캔과 자동으로 일치** — "직접 지은 티"의 절반은 텍스처 문제.

**P2-B. 한옥 부재 키트 v2 (지오메트리는 단순, 디테일은 트림시트)** — 공수 중~고
- 게임업계 관행: 디테일은 지오메트리가 아니라 **트림시트(부재 디테일을 그린 띠 텍스처)**가 담당.
- hall()/gate() 업그레이드 목록: ① 공포(栱包) 띠 — 처마 밑에 공포 사진 기반 트림시트 ② 서까래 밑면 방사 무늬
  ③ 창호 격자 면 ④ 기둥 배흘림 + 주춧돌 ⑤ 지붕 내림마루·추녀마루(대각 능선 4개) ⑥ 용마루 양성 마감.
- 재료는 P2-A의 스캔 크롭에서 나옴(공포·창호는 실측 스캔에 다 찍혀 있음).

**P2-C. 국가유산청 무료 게임 에셋 투입 (조사의 알짜 발견)** — 공수 저
- 국가유산청이 스케치팹·유니티·언리얼에 **무료 공개한 궁궐 소품 에셋**(어좌, 품계석, 병풍, 드므, 서책 등, 누적 100만 다운로드).
- 라이선스(출처표시 조건) 확인 후 기존 GLB 파이프라인으로 투입: 상자였던 품계석을 진짜 품계석으로, 근정전 앞 드므, (실내용) 어좌.

**P2-D. 순서 원칙**
- 근정전 본체의 최우선 해법은 여전히 **이용신청 승인 → 실측 스캔**. P2-B는 스캔이 영영 없는 건물(초기 시대 추정 복원 포함)용.
- 착수 순서: P2-A(텍스처) → P2-C(소품) → P2-B(부재 키트). 텍스처만 바꿔도 절반 해결이 조사 결론.

**P2-E. 조명·후처리 — AC 퀄리티의 세 번째 기둥 (기존 플랜에 빠져 있던 것)** — 공수 중, 효과/공수 비율 최상
- 진단: AC 화면이 사진 같은 이유의 절반은 모델이 아니라 **빛**(구운 전역 조명, 하늘빛이 재질에 스며듦).
  우리는 태양+반구광 2개뿐이라 좋은 스캔도 붕 떠 보인다.
- ① **IBL 환경광**: 시대별 하늘을 환경광 맵으로 변환(three.js PMREM + `scene.environment`) —
  모든 PBR 재질에 하늘빛·바닥 반사광이 스며듦. 시대 전환 시 환경맵도 교체(이미 시대별 하늘 텍스처 있음 → 그걸 소스로, 외부 파일 불필요·CSP 안전).
- ② **후처리 체인**(EffectComposer, 전부 내장 가능): SSAO(구석·처마 밑 자연 어둠 — 현 가짜 AO 띠의 상위 호환) + 은은한 Bloom(햇빛 면 하이라이트) + SMAA(계단 현상 제거). 1929 비네트·그레인(P1-C)도 이 체인에 통합.
- ③ 성능 가드: 후처리는 데스크톱 기본 켬 / 모바일·저사양은 자동 끔(프레임 측정 기반). SSAO 해상도 하프.
- 순서 주의: P2-E를 먼저 하면 P1-A(텍스처 파생)의 체감이 배가되므로 **v0.5로 앞당김**.

**히어로 동선 원칙 (AC의 실제 예산 배분 방식)**
- AC도 전 구역을 최고 퀄로 만들지 않는다 — 플레이어 동선만 진하게, 이면은 껍데기.
- 우리의 히어로 동선 = **광화문 → 흥례문 → 영제교 → 근정문 → 근정전 (+근정전 실내)**.
  4기둥(스캔·키트·조명·밀도) 투자를 이 축선에 집중하고, 서측(경회루 제외)·북측 침전 권역은 현 수준 유지.
  모든 v0.5~v0.7 작업 항목은 "히어로 동선에 해당하는가"로 우선순위를 판정한다.

## 문제 3 — "설명이 없다 · 실내가 없다 · 사람이 없다"

### P3-A. 건물 핫스팟 (구글 아트앤컬처 "i" 패턴)** — 공수 중
- 건물 위 "ⓘ" 마커 → 클릭 시 카드: 이름(한자) / 이름의 뜻 / 한 줄 일화 / 옛 사진·실록 링크 / 고증 등급.
- 1차 10동: 광화문·흥례문·영제교·근정문·근정전·사정전·강녕전·교태전·경회루·수정전.
- ⚠️ 콘텐츠 저작권: 궁능유적본부 "경복궁 이야기" 페이지는 **공공누리가 아니라 All rights reserved 확인됨** —
  문장 복제 금지. 역사적 사실(저작권 없음)만 참고해 자체 문장으로 재작성 + 실록 직접 인용(이미 확보한 발췌 활용).

### P3-B. 가이드 투어 — "왕의 하루" (AC 디스커버리 투어 구조)** — 공수 중
- 정거장 5개: ①광화문(궁의 스케일과 이름) → ②흥례문(궁으로 들어가는 절차) → ③근정전("부지런한 정치" — 정도전 작명, 1887 진하례 재현 지점) → ④사정전(매일 아침 정무) → ⑤강녕전(임금의 밤).
- 정거장당 60~90초: 자막 내레이션(사관(史官) 페르소나) + 관련 옛 사진/실록 인용 줌인 + 다음 정거장으로 glideTo.
- AC 방식 준수: **언제든 이탈 가능**, 정거장 간 즉시 이동, 총 7~10분(수업 1교시 안에 끝나는 길이 — 교육시장 겨냥).
- 기술적으로 새 시스템 불필요: 기존 카드 + glideTo 체인 + 대본 데이터만.

### P3-C. 실내 진입 — 근정전 1곳부터** — 공수 중~고
- 업계 표준은 파노라마지만, 우리는 걷는 3D이므로 **간이 3D 룸** 우선 검토: 어좌(P2-C 무료 에셋) + 일월오봉도(평면, 이미지 소스는 e뮤지엄/고궁박물관 공공누리 확인 필요) + 천장 쌍룡(카드 설명으로 대체 가능).
- 문 통과 시 페이드 + 노출 전환(어두운 실내로 눈이 적응하는 연출 — 업계 관행 확인).
- 근정전 본체 스캔 승인 시 내부 포함 여부 먼저 확인(우리 스캔들에 "_in" 내부 텍스처가 이미 있었음 — 동행각·강녕전은 내부도 찍혀 있다는 뜻).

### P3-D. 사람 — "반차도 실루엣" (고증 부담을 미학으로 뒤집기)** — 공수 중
- 조사 결론: 웹에서 군중은 빌보드(그림 팻말 방식)가 압도적으로 싸고, 실사 인물은 고증 부담이 큼.
- 우리 답: **의궤 반차도(왕실 행렬 기록화) 화풍의 실루엣 빌보드** — 선묘+담채 톤의 납작한 인물.
  ① 고증 논란 회피("기록화 인용"이라는 태도 자체가 고증적) ② 시각적 정체성이 됨 ③ 성능 부담 최소.
- 배치: 1888 근정전 마당 = 진하례 도열(품계석 열 맞춰, 실록 기록 기반) 15~20명 / 2026 = 관광객 5~8명 / 1929 = 박람회 관람객 5~8명 / 폐허 = 0명(비어있음이 연출).
- 상한 200장(성능 조사 기준), 스킨드메시(움직이는 3D 인체)는 1차 배제.
- 소스: 국립중앙박물관 외규장각 의궤 DB(museum.go.kr/uigwe)에 반차도 이미지 존재 — **라이선스 확인 필요**(미확인). 대안: 직접 그린 반차도풍 실루엣(저작권 원천 해결).
- 소리 레이어(사람보다 싸고 효과 큼): 시대별 앰비언스 — 1888 조회 웅성·발소리, 1929 확성기·군중, 2026 관광객·안내방송, 폐허 바람·까마귀. 공간감 있는 짧은 루프.
